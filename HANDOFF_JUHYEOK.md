# 박주혁 담당 업무 인수인계서

## 1. 담당 목표

참가자가 별도의 설명 없이도 프로그램을 시작하고, 자신의 참가자 ID를 입력한 뒤 테스트를 진행하고, 마지막에 개인 결과와 순위를 확인할 수 있는 전체 사용자 흐름을 구현한다.

오늘은 TASK 1과 TASK 2가 연결된 흐름까지만 완성한다. TASK 3과 TASK 4 화면은 실제 기능 대신 준비 중 화면 또는 빈 화면 구조만 만들어도 된다.

## 2. 담당 파일

주 담당 파일은 다음과 같다.

```text
ui/screens.py
ui/widgets.py
dashboard.py
```

필요하면 UI 관련 이미지와 글꼴을 `assets/`에 추가할 수 있다.

다음 파일은 다른 담당자와 먼저 상의하지 않고 수정하지 않는다.

```text
data_manager.py
scoring.py
analysis/
tasks/task1_reaction.py
tasks/task2_memory.py
```

`main.py`와 `task_manager.py` 연결이 필요하면 홍동열과 함께 수정한다.

## 3. 구현할 화면 흐름

```text
시작 화면
→ 참가자 ID 입력
→ ID 확인
→ 전체 테스트 안내
→ TASK 1 안내
→ TASK 1 진행 화면
→ TASK 1 임시 결과
→ TASK 2 안내
→ TASK 2 진행 화면
→ TASK 2 임시 결과
→ 개인 최종 결과
→ 현재 순위
→ 처음으로 돌아가기
```

## 4. 시작 화면과 참가자 ID 입력

### 화면 구성

시작 화면에는 다음 요소가 있어야 한다.

- 프로젝트 제목
- 한 줄 안내문
- 참가자 ID 입력칸
- `시작하기` 버튼
- 입력 오류 메시지 영역

권장 안내 문구:

```text
MODI 인지 능력 체험에 오신 것을 환영합니다.
결과 확인에 사용할 참가자 ID를 입력해 주세요.
실명 대신 기억하기 쉬운 별명 사용을 권장합니다.
```

### ID 입력 규칙

- 길이: 2~16자
- 허용 문자: 한글, 영문, 숫자, `_`, `-`
- 입력 앞뒤 공백은 자동 제거
- 빈 값은 허용하지 않음
- 이미 사용된 ID는 허용하지 않음
- 비밀번호나 전화번호 등의 개인정보 입력을 요구하지 않음
- Enter 키 또는 `시작하기` 버튼으로 제출

### 오류 안내

상황에 따라 다음과 같이 안내한다.

```text
ID를 입력해 주세요.
ID는 2~16자로 입력해 주세요.
사용할 수 없는 문자가 포함되어 있습니다.
이미 사용 중인 ID입니다. 다른 ID를 입력해 주세요.
```

### 데이터 담당과 연결

ID 입력 화면은 직접 CSV를 읽거나 쓰지 않는다. 데이터 담당자가 제공하는 함수를 호출한다.

예상 인터페이스:

```python
is_participant_id_available(participant_id: str) -> bool
create_participant_session(participant_id: str) -> dict
```

함수가 아직 준비되지 않았다면 임시 함수로 UI를 완성하고, 나중에 실제 함수로 교체한다.

세션 생성에 성공하면 다음 값을 이후 모든 화면에 전달한다.

```python
{
    "participant_id": "입력한 ID",
    "session_id": "데이터 담당자가 생성한 세션 ID"
}
```

## 5. 전체 테스트 안내 화면

다음 내용을 간단하게 표시한다.

- 총 네 가지 테스트가 있다는 설명
- 현재는 TASK 1과 TASK 2가 연결된 상태임을 개발 중에만 표시
- 테스트 중 장치를 임의로 분리하지 말라는 안내
- 연습 시행은 점수에 포함되지 않는다는 설명
- `테스트 시작` 버튼

실제 부스 운영 버전에서는 개발 중 문구를 제거한다.

## 6. TASK 안내 화면

각 TASK 시작 전에 다음 정보를 표시한다.

- TASK 번호와 이름
- 무엇을 해야 하는지
- 사용할 MODI 모듈
- 연습 여부
- `연습 시작` 또는 `테스트 시작` 버튼

### TASK 1 안내 문구

```text
빛과 소리가 나오면 현재 안내된 방식으로 빠르게 반응하세요.
버튼 조건에서는 버튼을 누릅니다.
자이로 조건에서는 손목을 아래쪽으로 한 번 뒤집습니다.
자극이 나오기 전에 움직이면 다시 측정합니다.
```

### TASK 2 안내 문구

```text
제시되는 색상 순서를 기억하세요.
다이얼을 돌려 색상을 선택하고 버튼을 눌러 확정합니다.
일부 시행에서는 색상과 함께 소리가 제시됩니다.
```

## 7. 테스트 진행 화면

진행 화면은 TASK의 측정 로직을 직접 구현하지 않는다. TASK 코드가 전달하는 상태를 화면에 표시한다.

표시해야 할 공통 정보:

- 참가자 ID
- 현재 TASK
- 연습 또는 측정 상태
- 현재 시행 번호 / 전체 시행 수
- 준비, 대기, 반응, 입력 완료 상태
- 중단 또는 센서 오류 메시지

예상 상태값:

```python
"instruction"
"practice"
"ready"
"waiting"
"stimulus"
"response"
"completed"
"error"
```

TASK 코드와 UI가 직접 강하게 결합되지 않도록, 상태 문자열과 화면에 표시할 메시지를 전달받는 방식으로 만든다.

## 8. 중간 결과 화면

TASK 1과 TASK 2가 끝난 직후 간단한 결과만 보여준다.

### TASK 1

- 버튼 대표 반응시간
- 자이로 대표 반응시간
- TASK 1 점수
- 다음 테스트 버튼

### TASK 2

- 시각 조건 정확도
- 시청각 조건 정확도
- 기억 폭
- TASK 2 점수
- 결과 보기 버튼

연구 지표를 좋음·나쁨으로 단정하는 표현은 사용하지 않는다.

## 9. 개인 최종 결과 화면

오늘은 TASK 1과 TASK 2 결과만 사용한 임시 결과 화면을 만든다.

표시 항목:

- 참가자 ID
- TASK별 점수
- 임시 종합점수
- 현재 참가자 중 순위
- 전체 참가자 수
- 각 영역의 원자료 요약

화면 하단에는 다음 문구를 표시한다.

```text
이 결과는 체험 및 탐구를 위한 간이 측정 결과이며,
의학적 진단이나 개인의 능력을 확정하는 자료가 아닙니다.
```

### 결과 데이터 형식

UI는 점수를 직접 계산하지 않고 데이터·점수 담당자가 제공한 결과를 표시한다.

예상 입력:

```python
{
    "participant_id": "brain01",
    "task_scores": {
        "task1": 82.5,
        "task2": 76.0,
        "task3": None,
        "task4": None
    },
    "total_score": 79.3,
    "rank": 3,
    "participant_count": 24,
    "metrics": {
        "button_median_ms": 310,
        "gyro_median_ms": 365,
        "visual_accuracy": 78.0,
        "audiovisual_accuracy": 84.0,
        "memory_span": 5
    }
}
```

## 10. 순위 화면

오늘은 상위 10명과 현재 참가자 위치만 표시한다.

표시 항목:

- 순위
- 참가자 ID
- 종합점수
- 현재 참가자 행 강조

동점 처리와 실제 순위 계산은 데이터·점수 담당자가 수행한다. UI에서는 전달받은 순위 목록만 그린다.

예상 입력:

```python
[
    {"rank": 1, "participant_id": "alpha", "total_score": 91.2},
    {"rank": 2, "participant_id": "modi7", "total_score": 88.5}
]
```

## 11. UI 공통 규칙

- 전체 화면 크기와 색상은 `config.py`의 설정값을 사용한다.
- 버튼, 제목, 안내문 스타일은 재사용 가능한 위젯으로 만든다.
- 한 화면에 너무 많은 글을 표시하지 않는다.
- 중요한 행동 버튼은 화면 오른쪽 아래 등 같은 위치에 둔다.
- 마우스뿐 아니라 Enter와 Escape 키도 지원한다.
- 센서 오류 시 프로그램이 종료되지 않고 안내 화면을 보여준다.
- 테스트 중에는 창 크기를 바꾸지 않는다.
- 글꼴이 없을 때도 한글이 깨지지 않는 기본 글꼴 대체 처리를 둔다.

## 13. 오늘 완료 기준

- [ ] 시작 화면이 표시된다.

- [ ] 잘못된 ID와 중복 ID 오류가 표시된다.
- [ ] 정상 ID 입력 후 참가자 세션이 생성된다.
- [ ] 전체 테스트 안내 화면으로 이동한다.
- [ ] TASK 1 안내·진행·중간 결과 화면이 연결된다.
- [ ] TASK 2 안내·진행·중간 결과 화면이 연결된다.
