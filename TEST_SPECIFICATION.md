# MODI 인지 능력 측정 시스템 테스트 명세서

## 1. 문서 목적

이 문서는 프로젝트에서 사용할 MODI 모듈, 네 가지 테스트의 진행 규칙, 저장할 데이터와 점수 계산법을 정의한다. 모든 참가자는 번호와 관계없이 동일한 조건, 시행 횟수, 조건 순서와 연습 기회를 제공받는다.

본 시스템은 체험 및 탐구용 프로토타입이다. 결과를 의학적 진단이나 개인의 지능·능력을 확정하는 자료로 사용하지 않는다.

## 2. 사용할 MODI 모듈

| 모듈 | 필수 여부 | 사용 과제 | 용도 |
|---|---:|---|---|
| 네트워크 | 필수 | 전체 | MODI와 노트북 연결 |
| 버튼 | 필수 | TASK 1, 2, 3, 4 | 반응, 선택 확정, 비트 입력 |
| 다이얼 | 필수 | TASK 2 | 색상 선택 |
| 자이로 | 필수 | TASK 1, 4 | 손목 동작 시작 감지, 커서 조작 |
| RGB LED | 필수 | TASK 1, 2, 3 | 시각 자극 및 색상 순서 제시 |
| 스피커 | 필수 | TASK 2, 3, 4 | 음높이 단서, 방해 자극, 비트 재생 |
| 디스플레이 | 선택 | 전체 | 간단한 상태·숫자 안내 |
| 모터 | 사용하지 않음 | - | 제공된 모터 모듈은 자체 진동 장치가 아니므로 제외 |

노트북의 pygame 화면은 전체 안내, 선택 상태, 커서, 결과와 그래프를 표시한다. MODI 디스플레이가 없거나 불안정하면 모든 안내를 노트북 화면으로 대체한다.

## 3. 전체 공통 규칙

1. 참가자에게 익명 ID를 부여한다.
2. 네 과제는 `TASK 1 → TASK 2 → TASK 3 → TASK 4` 순서로 수행한다.
3. 모든 참가자에게 같은 설명, 연습 횟수, 시행 수와 조건 순서를 적용한다.
4. 무작위 자극이 필요한 경우 참가자 ID에서 생성한 난수 시드를 저장한다.
5. 장치, 음량, LED 밝기, 화면 거리와 센서 위치를 부스 운영 중 동일하게 유지한다.
6. 시행별 원자료를 저장한 뒤 참가자 결과를 계산한다.
7. 연습 시행은 저장할 수 있지만 점수와 집단 통계에서는 제외한다.
8. 센서 연결 오류나 운영자 중단으로 실패한 시행은 `technical_error`로 기록하고 다시 시행한다.

## 4. TASK 1 - 반응속도와 입력 방식

### 4.1 측정 목적

같은 시각·청각 자극에 대해 버튼을 누르는 조건과 자이로를 움직이는 조건의 반응 시작시간을 측정한다. 두 입력 장치의 물리적 동작 시간이 다르므로 조건별 기록을 따로 표준화하며, 원시 시간의 차이는 입력 방식에 따른 반응 차이로만 해석한다.

### 4.2 사용 모듈

- RGB LED
- 스피커
- 버튼
- 자이로

### 4.3 자이로 동작

참가자는 자이로를 한 손에 수평으로 들고 기다린다. 자극이 나오면 손목을 한 번에 아래쪽으로 빠르게 뒤집는다.

- 반응 시작: 자극 후 자이로 각속도의 절댓값이 기준값을 처음 초과한 시점
- 동작 완료: 시작 자세보다 pitch 또는 roll이 40도 이상 변한 시점
- 시작 자세 복귀: 다음 시행 전 각도 차이가 10도 이내인 상태

각속도 기준값은 예비실험으로 결정하며 초기 권장값은 `40°/s`이다. 반응시간 비교에는 동작 완료가 아닌 반응 시작 시각을 사용한다.

### 4.4 진행 규칙

1. 버튼 1회와 자이로 1회를 연습한다.
2. 매 시행 화면에 준비 상태를 표시한다.
3. 1.5~3.0초의 무작위 대기 후 RGB LED와 스피커 자극을 동시에 제시한다.
4. 참가자는 현재 조건에 따라 버튼을 누르거나 자이로를 움직인다.
5. 자이로 조건에서는 센서가 시작 자세로 돌아온 뒤 다음 시행을 시작한다.
6. 측정 시행은 아래의 고정 순서로 총 8회 진행한다.

```text
버튼 → 자이로 → 자이로 → 버튼 → 자이로 → 버튼 → 버튼 → 자이로
```

### 4.5 유효 시행 기준

- 자극 전 입력: 조기 반응으로 제외 후 재시행
- 150ms 미만: 예측 반응으로 제외 후 재시행
- 1,500ms 초과: 지연 반응으로 기록하되 대표값 계산에서 제외
- 조건별 유효 시행이 3회 미만: 부족한 수만큼 재시행

### 4.6 기록 항목

- participant_id
- trial_index
- input_condition: `button` 또는 `gyro`
- random_wait_ms
- stimulus_time
- response_start_time
- response_complete_time
- reaction_time_ms
- completion_time_ms
- gyro_pitch, gyro_roll
- gyro_angular_velocity
- early_response
- delayed_response
- valid
- retry_count

### 4.7 점수 계산

조건별 유효 시행의 중앙값을 대표 반응시간으로 사용한다.

```text
버튼 대표값 = 버튼 유효 반응시간의 중앙값
자이로 대표값 = 자이로 유효 반응 시작시간의 중앙값
입력 방식 차이 = 자이로 대표값 - 버튼 대표값
```

버튼과 자이로는 각각 참가자 집단 안에서 빠른 기록이 높은 점수를 받도록 백분위 점수로 변환한다.

```text
TASK 1 점수 = (버튼 백분위 점수 + 자이로 백분위 점수) / 2
```

참가자가 10명 미만일 때는 팀 예비실험 자료를 임시 기준 집단으로 사용한다. 참가자가 10명 이상 모이면 전체 참가자 자료로 백분위와 순위를 다시 계산한다. 입력 방식 차이는 연구용 지표이며 좋고 나쁨을 나타내는 점수에는 포함하지 않는다.

## 5. TASK 2 - 기억력과 감각 통합

### 5.1 측정 목적

색상만 제시하는 시각 조건과 색상·음높이를 함께 제시하는 시청각 조건에서 순서 기억의 정확도와 기억 폭을 비교한다.

### 5.2 사용 모듈

- RGB LED
- 스피커
- 다이얼
- 버튼

### 5.3 자극 대응

| 색상 | 음높이 예시 | 다이얼 구간 |
|---|---:|---:|
| 빨강 | 262Hz(C4) | 0~24 |
| 초록 | 330Hz(E4) | 25~49 |
| 파랑 | 392Hz(G4) | 50~74 |
| 노랑 | 523Hz(C5) | 75~100 |

각 자극은 600ms 동안 제시하고 자극 사이에는 300ms의 공백을 둔다. 시청각 조건에서는 LED와 음을 동시에 시작하고 종료한다.

### 5.4 진행 규칙

1. 다이얼로 색상을 선택하고 버튼으로 확정하는 방법을 한 번 연습한다.
2. 시각 조건과 시청각 조건에서 각각 길이 3, 4, 5, 6의 순서를 한 번씩 제시한다.
3. 아래와 같이 같은 길이의 두 조건을 연속 배치하되, 먼저 나오는 조건을 길이마다 교차한다.

| 시행 | 조건 | 순서 길이 |
|---:|---|---:|
| 1 | 시각 | 3 |
| 2 | 시청각 | 3 |
| 3 | 시청각 | 4 |
| 4 | 시각 | 4 |
| 5 | 시청각 | 5 |
| 6 | 시각 | 5 |
| 7 | 시각 | 6 |
| 8 | 시청각 | 6 |

4. 색상 순서는 참가자별 난수 시드로 생성한다.
5. 같은 색상이 세 번 이상 연속되지 않도록 제한한다.
6. 참가자는 다이얼로 색상을 선택하고 버튼으로 각 항목을 확정한다.
7. 입력 수정 기능은 제공하지 않는다.

### 5.5 기록 항목

- participant_id
- trial_index
- sensory_condition: `visual` 또는 `audiovisual`
- sequence_length
- target_sequence
- response_sequence
- position_correct_count
- exact_sequence_correct
- response_start_time
- response_complete_time
- total_response_time_ms
- random_seed

### 5.6 점수 계산

각 조건에서 위치 정확도와 기억 폭을 계산한다.

```text
위치 정확도 = 맞힌 위치 수의 합 / 제시된 전체 항목 수 × 100
기억 폭 = 완전히 맞힌 가장 긴 순서 길이
기억 폭 점수 = 기억 폭 / 6 × 100
조건 점수 = 위치 정확도 × 0.7 + 기억 폭 점수 × 0.3
TASK 2 점수 = (시각 조건 점수 + 시청각 조건 점수) / 2
감각 통합 효과 = 시청각 위치 정확도 - 시각 위치 정확도
```

감각 통합 효과는 연구용 비교값이며 종합점수에는 별도로 가산하지 않는다.

## 6. TASK 3 - 선택적 주의와 감각 모순

### 6.1 측정 목적

참가자가 시각 자극에만 반응하고 청각 자극을 억제하도록 하여 목표 탐지 능력, 반응 억제와 청각 간섭을 측정한다. 횟수 카운팅은 작업기억의 영향을 크게 받으므로 사용하지 않는다.

### 6.2 사용 모듈

- RGB LED
- 스피커
- 버튼

### 6.3 참가자 지시

```text
빛이 나오면 버튼을 누르세요.
소리만 나오면 누르지 마세요.
```

### 6.4 진행 규칙

1. 빛+소리 1회와 소리만 1회를 연습한다.
2. 한 시행의 자극은 150ms 동안 제시한다.
3. 자극 후 700ms 동안 버튼 입력을 기다린다.
4. 시행 사이 대기시간은 0.8~1.5초 중 무작위로 정한다.
5. 총 16회를 아래의 고정된 조건 순서로 진행한다.

기호는 `C=빛+소리`, `V=빛만`, `A=소리만`, `N=무자극`을 뜻한다.

```text
C → V → A → C → N → A → V → C → A → C → V → A → C → N → V → C
```

조건별 시행 수는 다음과 같다.

- 빛+소리: 6회
- 빛만: 4회
- 소리만: 4회
- 무자극: 2회

### 6.5 판정 기준

- 빛+소리 또는 빛만 조건에서 150~700ms 사이 버튼 입력: 적중
- 빛이 나왔지만 입력하지 않음: 누락
- 소리만 또는 무자극 조건에서 버튼 입력: 오경보
- 자극 후 150ms 미만 입력: 예측 반응으로 기록하고 오경보 처리

### 6.6 기록 항목

- participant_id
- trial_index
- stimulus_condition: `congruent`, `visual_only`, `audio_only`, `none`
- random_wait_ms
- stimulus_time
- button_time
- reaction_time_ms
- hit
- miss
- false_alarm
- correct_rejection
- anticipatory_response

### 6.7 점수 계산

```text
적중률 = 적중 수 / 빛이 포함된 시행 수 × 100
억제 성공률 = 정확히 누르지 않은 시행 수 / 빛이 없는 시행 수 × 100
균형 정확도 = (적중률 + 억제 성공률) / 2
청각 간섭률 = 소리만 조건의 오경보 수 / 소리만 조건 수 × 100
```

빛이 포함된 정답 시행의 반응시간 중앙값을 집단 내 백분위로 변환한다.

```text
TASK 3 점수 = 균형 정확도 × 0.8 + 반응시간 백분위 점수 × 0.2
```

정확도가 50% 미만이면 반응시간 점수는 0점으로 처리한다. 청각 간섭률은 연구용 지표로 별도 표시한다.

## 7. TASK 4 - 운동 협응과 이중 과제

### 7.1 측정 목적

자이로로 커서를 목표 안에 유지하면서 스피커 비트에 맞춰 버튼을 누르는 능력을 측정한다. 단일 과제 기준값과 이중 과제 수행을 함께 기록한다.

### 7.2 사용 모듈

- 자이로
- 스피커
- 버튼
- 노트북 화면

### 7.3 진행 규칙

1. 자이로 커서 조작을 5초간 연습한다.
2. 비트 버튼 입력을 4회 연습한다.
3. 자이로 단독 기준 측정을 10초간 진행한다.
4. 버튼 단독 기준 측정을 8비트 동안 진행한다.
5. 자이로와 버튼을 동시에 사용하는 이중 과제를 20초간 진행한다.
6. 비트 간격은 750ms로 고정한다.
7. 커서 목표는 방향과 속도가 구간마다 달라지는 10초 고정 경로를 연속 이동한다.
8. 자이로 단독 조건에서는 경로를 1회, 이중 과제 조건에서는 같은 경로를 2회 반복한다.
9. 매 시작 전 1초 동안의 자이로 평균값을 중립 위치로 보정한다.

### 7.4 커서와 버튼 판정

- 자이로 pitch와 roll을 커서의 Y축과 X축 이동으로 변환한다.
- 커서 중심이 현재 이동 목표 원 내부에 있으면 목표 유지로 판정한다.
- 각 비트에서 가장 가까운 버튼 입력 하나만 대응시킨다.
- 비트 전후 400ms 이내 입력을 유효 입력으로 인정한다.
- 대응되지 않은 버튼 입력은 추가 입력으로 기록한다.

### 7.5 기록 항목

- participant_id
- phase: `cursor_baseline`, `rhythm_baseline`, `dual_task`
- sample_time
- gyro_pitch, gyro_roll
- cursor_x, cursor_y
- distance_from_target
- inside_target
- beat_index
- beat_time
- button_time
- beat_error_ms
- missed_beat
- extra_press
- calibration_pitch, calibration_roll

### 7.6 점수 계산

```text
커서 유지율 = 목표 안에 있던 샘플 수 / 전체 샘플 수 × 100
평균 비트 오차 = 유효 입력의 |버튼 시각 - 비트 시각| 평균
리듬 정확도 = max(0, 100 - 평균 비트 오차 / 4)
누락·추가 감점 = min(30, (누락 수 + 추가 입력 수) × 5)
보정 리듬 점수 = max(0, 리듬 정확도 - 누락·추가 감점)
TASK 4 점수 = 이중 과제 커서 유지율 × 0.5 + 이중 과제 보정 리듬 점수 × 0.5
```

이중 과제 비용은 연구용으로 별도 계산한다.

```text
커서 이중과제 비용 = 단독 커서 유지율 - 이중과제 커서 유지율
리듬 이중과제 비용 = 이중과제 평균 비트 오차 - 단독 평균 비트 오차
```

## 8. 종합점수와 순위

네 과제는 같은 비중으로 반영한다.

```text
종합점수 = TASK 1 × 0.25 + TASK 2 × 0.25 + TASK 3 × 0.25 + TASK 4 × 0.25
```

- 종합점수는 소수점 첫째 자리까지 표시한다.
- 동점이면 TASK 2, TASK 3, TASK 4, TASK 1 순서의 점수를 차례로 비교한다.
- 백분위가 포함된 TASK 1과 TASK 3 순위는 참가자가 추가될 때 다시 계산될 수 있다.
- 체화 반응 차이, 감각 통합 효과, 청각 간섭률과 이중 과제 비용은 좋고 나쁨을 단정하지 않고 개인·집단 분석값으로 제시한다.

## 9. 예비실험에서 확정할 값

다음 값은 본 운영 전 팀원 예비실험을 통해 최종 보정한다.

- 자이로 반응 시작 각속도 기준값
- 자이로 동작 완료 각도
- TASK 3 입력 허용시간 700ms
- TASK 4 커서 민감도와 목표 원 크기
- 스피커 음량과 LED 밝기
- 전체 체험 소요시간

보정 후 변경된 값은 `config.py`에 기록하고 프로그램 버전과 함께 저장한다.
